# 电商AI助手 - 项目规范

## 一、项目概述

**项目名称**: 电商AI助手 (E-commerce AI Assistant)
**类型**: Windows桌面应用
**核心功能**: 连接大语言模型，对电商数据进行选品分析、热词分析、带货数据分析
**目标用户**: 电商从业者、直播带货主播、选品专员

---

## 二、技术架构

### 2.1 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 桌面框架 | Tauri 2.x | Rust后端 + Web前端，打包体积小 |
| 前端框架 | Vue3 + Vite + TypeScript | 现代化Web开发 |
| UI组件 | Element Plus | 成熟的后台管理UI |
| 状态管理 | Pinia | Vue3官方推荐 |
| HTTP客户端 | ky / fetch | 调用LLM API |
| 本地存储 | Tauri fs API | 读写本地数据文件 |

### 2.2 支持的LLM模型

| 模型 | API格式 | 说明 |
|------|---------|------|
| OpenAI (GPT-4/GPT-3.5) | OpenAI Chat Completions | 通用 |
| Anthropic Claude | Anthropic Messages | 需单独API |
| MiniMax | MiniMax Chat Completions | 国内可用 |
| Kimi (Moonshot) | Moonshot API | 国内可用 |
| 硅基流动 (SiliconFlow) | OpenAI兼容 | 国内可用 |
| Ollama 本地模型 | Ollama API | 本地部署 |

## 三、功能模块

### 3.1 LLM对话模块

**功能描述**: 支持多模型切换的通用对话界面

**支持的操作**:
- 选择LLM模型（MiniMax / Kimi / OpenAI / Ollama等）
- 输入API Key并保存到本地
- 发送消息，接收流式/非流式回复
- 对话历史保存到本地
- 预设Prompt模板（选品助手、热词分析、带货分析）
- 导出对话记录

**数据流**:
```
用户输入 → 选择Prompt模板 → 选择模型 → 调用API → 渲染回复
```

**UI布局**:
```
┌─────────────────────────────────────────┐
│  [模型选择 ▼] [API Key输入框] [保存]     │
├─────────────────────────────────────────┤

├─────────────────────────────────────────┤

│                                         │
```

- 纯文本粘贴
- JSON格式数据

**分析维度**:
- 市场容量评估
- 竞争度分析（同类商品数量/价格分布）

- 热词分类（品类词、品牌词、场景词、功效词）

### 3.4 带货数据分析模块

**分析维度**:
- GMV/UV价值评估
- 转化率漏斗分析（曝光→点击→加购→付款）

- 互动率评估
- 与竞品对比
- 优化建议

```
3. 与行业均值对比
4. 问题诊断
5. 优化建议

{user_data}
```

## 五、数据安全

- API Key使用本地加密存储，不上传任何服务器
- 对话历史保存在本地Tauri数据目录
- 支持数据导入导出，不强制云端同步

## 六、打包部署

### 6.1 目标平台
- Windows 10/11 (x64)
- 生成单个 .exe 安装包

### 6.2 打包配置
- 使用 Tauri bundler
- 安装包命名: `电商AI助手_v1.0.0_x64.exe`
- 开启 CSP 安全策略
- 应用图标自定义

## 七、验收标准

### 功能验收
- [ ] 可以切换不同LLM模型进行对话
- [ ] 可以导入CSV/TXT/JSON本地数据进行选品分析
- [ ] 可以导入热词数据进行热词分析
- [ ] 可以导入带货数据进行数据分析
- [ ] 分析结果可以复制/导出
- [ ] 对话历史可以保存和加载

### 性能验收
- [ ] 冷启动时间 < 3秒
- [ ] API响应在模型返回后立即展示
- [ ] 本地数据处理 < 1秒

### 打包验收
- [ ] 生成可运行的Windows安装包
- [ ] 安装包体积 < 50MB
- [ ] 安装后可以正常运行
